sales

get sales

Retrieve sales data with pagination support. Customer filter required: When using page-based pagination (page parameter), you must provide either customerSourceId or customerTilroyId. When using lastId-based pagination (lastId parameter), customer filters are optional.

query Parameters
count
string
Default: "10"

max number of sales to be returned (default = 10, max = 100)

page
string
Default: "1"

page of results ot be returned (default = 1)

lastId
string
Examples:
  • lastId= - Start cursor-based pagination
  • lastId=1230010033307 - Continue with specific sale ID

Enables cursor-based pagination using sale IDs. Use empty value (?lastId=) to start cursor pagination, then use the last sale ID from previous response for subsequent requests. Cannot be used together with page parameter. When used, only Link and Link-JSON headers are returned (no X-Paging-* headers).

fields
string

filter out what you want returned for each sale by specifying one or more properties separated by a comma (ex. fields=idTilroySale,customer,shop.name)

sorting
string

this is a comma seperated list in which you can define what fields you to sort by. Syntax is: 'sorting=field1:1,field2:-1,...'. Asc/Desc is specified by the value sign, 1 for asc., -1 for desc.

customer
string
Deprecated

filter results by customer sourceId (required when using page-based pagination, optional when using lastId pagination)

customerSourceId
string

filter results by customer sourceId (required when using page-based pagination, optional when using lastId pagination)

customerTilroyId
string

filter results by customer tilroyId (required when using page-based pagination, optional when using lastId pagination)

dateFrom
string

filter results by daterange (must be accompanied by dateTo), dates are in ISO format and only days are taken into account

dateTo
string

filter results by daterange (must be accompanied by dateFrom), dates are in ISO format and only days are taken into account

orderNumber
string

filter results by order number, requires customerTilroyId

excludeInvoiceLinks
boolean
Default: false

exclude invoice.links array from the response to improve response times - when true, the invoice.links property will not be populated (default: false)
