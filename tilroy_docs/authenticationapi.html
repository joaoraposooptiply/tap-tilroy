password

Update the password of a login

Use this endpoint to change the password of a login.

Changing a password will also invalidate the current user's refresh token! After a password change, a user must login by username and password first, to get a new valid refresh token.

hashes

Generates a hash to identify a customer

This endpoint can be used for 2 uses:

  1. It should be called to receive a hash to reset the users password. When this hash is send to the provided email address, it is possible to authenticate the user. For this use, you need to provide the type 'login' (default)
  2. It should be called to receive a hash to un/subscribe a customer. After sending an url with this hash in an email to the customer, it is possible to identify the customer to unsubscribe.. For this use, you need to provide the type 'customer'

A 'login' hash has a shorter lifespan, then a 'customer' hash. Creating a 'customer' hash, will not invalidate a 'login' hash or vice versa
