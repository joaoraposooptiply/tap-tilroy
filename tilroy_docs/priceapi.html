import

Import prices

Import prices into Tilroy.

Shops

When both providing shopNumber and idSourceShop, matching on idSourceShop will get priority over shopNumber.

Runs

When you pass in a runId that does not yet exist in Tilroy, that run will be created in Tilroy, and future calls will match to that run.

Price Types

Either price or volumePrices must be filled in.

prices

Get current price for multiple skus

This endpoint acts similar to GET /prices/{skuId}/shop/{shopId}, but is meant to be used when you want the current price for multiple skuIds. Either skuIds or skuSourceIds must be passed

query Parameters
skuIds
required
string

Comma-separated list of all skuIdentifiers you want a price for. This is limited to a maximum of 100 ids.

skuSourceIds
required
string

Comma-separated list of all sku sourceIds you want a price for. This this takes priority over skuIds passed.

shopId
string

Filter the result on all prices for this sku and this shop. (shopId or shopNumber required)

shopNumber
string

Filter the result on all prices for this sku and this shop. (shopId or shopNumber required)

countryCode
string

Filter the result on all prices for this sku and this country. Should be a standard 2-letter ISO code. (Shop country will be used if not available)

priceRange
string
Default: "retail"

Determine what price range to use for this call ("retail", "wholesale" or a custom range).

date
string <date-time>

Datetime (ISO 8601) to get prices on a specified point in time (for future or historic prices, you don't need to use this to get the current price) - 2020-01-01T00:00:00Z

fields
string

Comma-separated list of all fields you want returned in the result. All other fields will be omitted.

header Parameters
Tilroy-Api-Key
required
string

API key to identify and validate the tenant

x-api-key
required
string

API key of AWS

Responses

https://api.tilroy.com/priceapi/production/prices

Get the current price for an sku Deprecated

Please use "/prices/{skuId}".

Get the current price for a given sku, consisting of at the most a standard price and a promo price.

Adding a filter on countryCode and / or shopId results in a price that matches the filter as closely as possible. This means that when you add a countryCode filter, the result will be a price for that country, when it is available. When you specify a shopId, the result will be a price for the shop, when it is available. When you pass in both countryCode and shopId, the result will fallback to a country-price (when one is found) when no shop-price could be found.

path Parameters
skuId
required
string

Identifier of the sku to get the price for.

shopId
required
string

Identifier of the shop to get the price for.

query Parameters
countryCode
string

Filter the result on all prices for this sku and this country. Should be a standard 2-letter ISO code. (Shop country will be used if not available)

priceRange
string
Default: "retail"

Determine what price range to use for this call ("retail", "wholesale" or a custom range).

date
string <ISO 8601>

Provide this when you want the price on a specific date(ISO 8601 format).

fields
string

Comma-separated list of all fields you want returned in the result. All other fields will be omitted.

Get all price rules

Get all price rules without requiring a specific sku. This endpoint returns detailed price rule information including shop, country, and price item details with optional filtering by shop and pagination support. Use either page-based pagination (page parameter) or cursor-based pagination (lastId parameter), but not both.

string

count
integer

page
