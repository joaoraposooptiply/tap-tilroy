CustomerAPI (2.33.1)

Download OpenAPI specification:

This is the public API for managing customer related data.

Security

In order to use this api, you'll need multipe headers:

  • Tilroy-Api-Key: The api key of the tenant, so that we know what data you are allowed to manage
  • x-api-key: The api key of AWS, so that we know, what you are allowed to do (access specific endpoints or specific throttling rules)
  • Authorization: The bearer token of the customer, so that we know if you are allowed of retrieving/updating data of a specific customer

If you are developping an app where the customer needs to update his/her own data, you can use the AuthenticationAPI to get a token for the customer by using the /tokens endpoint.

If you are writing a service for syncing customer data, we will need to provide you a bearer token with access to allow retrieving/updating the data of all customers.

When you get an error 'User xxx has no permission to update customer' you are probably using a wrong combination of headers. Both the Tilroy-Api-Key and the the bearer token should be linked to the same tenant. This can happen when you have multiple tenants for each environment you are integrating with.
