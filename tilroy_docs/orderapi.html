search orders

Use this endpoint to search for orders. At least a customerId or a shopId should be provided (both can provided in one call as well).

The query parameter "basketId" can contain a comma-separated list of basketIds.

Because of performance reasons the response header "X-Paging-ItemCount" is not always correct. It is only correct on the last page.
This is why you will have to rely on the response-header "Link-JSON" if you want to fetch all results. Only if the "next" properyt of "Link-JSON" is empty, the last page is reached.

count
string
Default: 10

max number of items to be returned (default = 10, max = 100)
